الفكرة ببساطة

قاعدة البيانات تحتفظ بملف اسمه Transaction Log يسجل كل العمليات التي تحدث، مثل:

إضافة بيانات (INSERT)

تعديل بيانات (UPDATE)

حذف بيانات (DELETE)

Transaction Log Backup هو أخذ نسخة احتياطية من هذا السجل.

لماذا نستخدمه؟ 🔹

استرجاع البيانات بدقة

يمكنك استرجاع قاعدة البيانات إلى لحظة معينة قبل حدوث مشكلة.

حماية البيانات

إذا حدث Crash أو مشكلة في السيرفر، يمكن استرجاع آخر التغييرات.

تقليل فقدان البيانات

بدل الاعتماد على Backup يومي فقط، يمكن أخذ Log Backup كل 10 أو 15 دقيقة.

مثال بسيط

لو عندك:

Full Backup الساعة 12:00

Log Backup الساعة 12:10

Log Backup الساعة 12:20

ولو حصلت مشكلة 12:25
يمكنك استرجاع البيانات حتى 12:20 تقريبًا.

متى يعمل؟

يعمل عندما يكون Recovery Model:

Full

Bulk-Logged

ولا يعمل مع:

Simple Recovery Model

✅ باختصار:
Transaction Log Backup = نسخة احتياطية من سجل التغييرات في قاعدة البيانات تساعدك ترجع البيانات لنقطة زمنية محددة.
===================================================================================
DELETE
أمر يستخدم لحذف صفوف محددة من الجدول.
مميزاته

يمكن حذف صفوف معينة باستخدام WHERE.

يتم تسجيل كل صف يتم حذفه في Transaction Log.

يمكن عمل Rollback إذا كان داخل Transaction.

يعمل مع الجداول التي تحتوي على Foreign Keys.

العيوب

أبطأ إذا كان الجدول كبير لأن الحذف يتم Row by Row.

TRUNCATE

أمر يستخدم لحذف كل البيانات من الجدول بالكامل.

مثال:

TRUNCATE TABLE Employees;
مميزاته

أسرع بكثير من DELETE.

لا يسجل حذف كل صف في Transaction Log بل يسجل إزالة الصفحات فقط.

يعيد Identity Column إلى البداية.

القيود

لا يمكن استخدام WHERE.

لا يعمل إذا كان الجدول مرتبط بـ Foreign Key.

يحذف كل البيانات مرة واحدة.
=========================================================================
CTE (Common Table Expression)
CTE هو جدول مؤقت بيتعمل داخل الاستعلام نفسه فقط.

🔹 بيتكتب قبل الـ SELECT باستخدام WITH.
🔹 بيعيش وقت تنفيذ الكويري فقط.
🔹 مفيد لتنظيم الكويري المعقدة.
🔹 ممكن تستخدمه في الاستعلامات recursive (الاستدعاء الذاتي).
====
View هو جدول افتراضي محفوظ في قاعدة البيانات.

🔹 بيتخزن في الداتابيز.
🔹 ممكن أي حد يستخدمه في أي وقت.
🔹 بيبسط الكويري الكبيرة ويخليها reusable.

الخلاصة:

استخدم CTE لو عايز تقسّم كويري كبيرة أو تعمل logic مؤقت.

استخدم View لو عايز تحفظ استعلام وتستخدمه كتير.
